==============
The Controller

The handheld controller is the primary human-interface device for the hexapod. It communicates
with the robot via the onboard antenna module, transmitting directional commands, speed
setpoints, and mode-selection signals in real time.

The controller was designed to be ergonomic and low-latency. A dedicated microcontroller on
the controller board samples the joystick axes and button states, packages them into a compact
command frame, and dispatches the frame at a fixed rate over the wireless link. On the robot
side, the main MCU decodes incoming frames and forwards the relevant setpoints to the gait
scheduler and IK solver.

Key design decisions for the controller include:

- **Compact command frame** — only the minimum required data is transmitted per cycle to reduce
  airtime and latency.
- **Heartbeat monitoring** — if the robot does not receive a valid frame within a configurable
  timeout window it enters a safe-stop state.
- **Mode selector** — a rotary switch allows the operator to toggle between manual drive,
  autonomous patrol, and camera-tracking modes.
